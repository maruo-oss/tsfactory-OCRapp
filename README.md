# AI-OCR 発注管理システム

Google Apps Script と Gemini AI を使用した、PDF発注書の自動OCR処理・管理システムです。

## 概要

このシステムは、PDF形式の発注書を自動的に読み取り、Google Spreadsheet に発注データを保存します。AI（Gemini 2.5 Pro）により発注書の内容を構造化データとして抽出し、Webインターフェースで確認・編集が可能です。

### 主な機能

- **AI-OCR自動処理**: Gemini API を使ってPDF発注書から情報を自動抽出（ページ単位で処理）
- **発注番号抽出**: 発注書から発注No（発注番号）を自動抽出・表示
- **拠点別管理**: Google Drive のフォルダ構造で複数拠点を管理
- **納品先対応**: 発注書から納品先情報を抽出・管理
- **Webインターフェース**: ブラウザ上で発注内容の閲覧・編集
- **PDFプレビュー**: 原本PDFをリアルタイム表示
- **ステータス管理**: 6段階のステータス（未処理/処理中/確認中/完了/保留/要確認）で進捗管理
- **アーカイブ機能**: 完了した発注書をアプリ内で非表示化
- **完了レコード自動削除**: 完了ステータスのレコードを毎日深夜に自動物理削除（トリガー設定）
- **ページタブ**: 複数ページPDFの商品明細をページ別タブで切り替え表示
- **商品明細ソート**: PDFの元の順序を保持して表示
- **商品明細行番号**: No.列で商品の順序を視覚的に表示（1, 2, 3...）
- **リサイザブルUI**: 発注詳細と商品明細の上下調整、左右パネル幅のドラッグ調整に対応
- **Excel出力**: 選択した発注書をまとめてExcel形式でエクスポート（出力後自動アーカイブ）
- **データ編集**: 発注日、メーカー、商品情報（発注No・品番・品名・数量）、ファイル名、備考などを手動修正可能
- **明細行削除**: 編集モードで不要な商品明細行を個別に削除可能（スプレッドシートから物理削除）

## システム構成

```
Google Drive (PDF保管)
    ↓
Google Apps Script (OCR処理)
    ├─ Gemini Flash: ページ数取得
    └─ Gemini Pro × ページ数: 明細抽出（ページごと）
    ↓
Google Spreadsheet (データ保存)
    ↓
Web UI (閲覧・編集)
```

## セットアップ

### 1. 必要なもの

- Google アカウント
- Gemini API キー（[Google AI Studio](https://makersuite.google.com/app/apikey) で取得）
- Google Drive のフォルダID × 2（入力用、処理済み用）
- Google Spreadsheet

### 2. Google Apps Script プロジェクトの作成

1. [Google Apps Script](https://script.google.com/) にアクセス
2. 新規プロジェクトを作成
3. 以下のファイルを作成・配置：
   - `Code.gs`: メインスクリプト
   - `index.html`: UI HTML
   - `appsscript.json`: プロジェクト設定

### 3. スクリプトプロパティの設定

スクリプトエディタで「プロジェクトの設定」→「スクリプト プロパティ」から以下を設定：

| プロパティ名 | 説明 | 例 |
|------------|------|-----|
| `GEMINI_API_KEY` | Gemini APIキー | `AIza...` |
| `ROOT_IN_FOLDER_ID` | 処理対象PDFを配置するフォルダID | `1Abc...` |
| `ROOT_PROCESSED_FOLDER_ID` | 処理済みPDFの移動先フォルダID | `1Def...` |
| `SPREADSHEET_ID` | データ保存先スプレッドシートID | `1Ghi...` |

#### フォルダ構造の例

```
📁 ROOT_IN_FOLDER (処理対象)
  ├─ 📁 東京
  │   ├─ 📄 発注書_001.pdf
  │   └─ 📄 発注書_002.pdf
  └─ 📁 大阪
      └─ 📄 発注書_003.pdf

📁 ROOT_PROCESSED_FOLDER (処理済み)
  ├─ 📁 東京 (自動作成)
  └─ 📁 大阪 (自動作成)
```

### 4. Drive APIの有効化

`appsscript.json` で既に設定済みです：
```json
"enabledAdvancedServices": [
  {
    "userSymbol": "Drive",
    "version": "v3",
    "serviceId": "drive"
  }
]
```

### 5. プロンプトシートの作成

スプレッドシートに「プロンプト」という名前のシートを作成し、A1セルに以下のようなプロンプトを記述：

```
以下のPDF発注書から以下の情報をJSON形式で抽出してください：
- order_number: 発注番号/発注No/注文番号（見つからない場合は空文字列）
- order_date: 発注日（YYYY-MM-DD形式）
- maker_name: メーカー名
- shop_name: 店舗名
- delivery_destination: 納品先（株式会社/有限会社など）
- items: 商品明細の配列
  - product_code: 品番
  - product_name: 商品名
  - quantity: 数量
  - unit_price: 単価
```

> **注意**: `page_number` はプロンプトに含めません。システムがPDFをページ単位で処理し、プログラム的にページ番号を付与します。

### 6. Webアプリとしてデプロイ

1. スクリプトエディタで「デプロイ」→「新しいデプロイ」
2. 種類: Webアプリ
3. 実行ユーザー: 自分
4. アクセスできるユーザー: 組織内（または必要に応じて設定）
5. デプロイを実行してURLを取得

## 使い方

### 発注書の自動処理

1. Google Drive の `ROOT_IN_FOLDER` 配下に拠点名フォルダを作成
2. 各拠点フォルダにPDF発注書を配置
3. Webアプリまたはスクリプトエディタから `processOrders()` を実行
   - Web UI: 「自動取込実行」ボタンをクリック
   - または Apps Script トリガーで定期実行

### Web UIの操作

#### 発注書の閲覧
1. 左サイドバーから発注書を選択
2. 中央ペインで詳細情報を確認
3. 右ペインでPDF原本をプレビュー

#### データの編集
1. 発注書を選択
2. 「編集」ボタンをクリック
3. 必要な項目を修正（ファイル名、発注日、メーカー、ステータス、備考、商品情報など）
4. 「保存」ボタンで変更を確定

#### 商品明細行の削除
1. 発注書を選択し、「編集」ボタンをクリック
2. 商品明細テーブルの各行左端に赤い「✕」ボタンが表示される
3. 削除したい行の「✕」ボタンをクリック → 確認ダイアログで「OK」
4. 行が非表示になる（この時点ではまだ削除されていません）
5. 「保存」ボタンで確定するとスプレッドシートから物理削除される
6. 「キャンセル」ボタンで削除を取り消し可能

#### 画面レイアウトの調整（リサイザー機能）
- **上下リサイザー**: 発注詳細と商品明細の間の灰色の区切り線をドラッグして上下に調整
- **左右リサイザー**: サイドバー・プレビューペインとの境界線をドラッグして幅を調整
- 最小高さ200px、サイドバー幅200〜600px、プレビュー幅200〜800pxの範囲で調整可能

#### アーカイブ機能（完了管理）
1. 処理が完了した発注書にチェック
2. 「選択した発注書を完了にする」ボタンをクリック
3. アプリ内では非表示になります
4. 完了レコードは毎日深夜2時に自動物理削除されます（トリガー設定時）

#### Excel出力と自動アーカイブ
1. 出力したい発注書にチェック
2. 「選択した店舗をExcel出力」ボタンをクリック
3. 確認ダイアログで「OK」をクリック
4. Excelファイルがダウンロードされ、自動的に「完了」ステータスになります
5. アプリ内では非表示になり、定期削除の対象になります

#### ページタブ（複数ページPDF対応）
1. 複数ページのPDF発注書を取り込むと、商品明細の上にページタブが表示されます
2. 各タブをクリックすると、そのページの商品のみが表示されます
3. 「全ページ」タブで全商品を一覧表示
4. 単一ページのPDFではタブは表示されません（従来通り）

#### フィルタリング
- ヘッダーの「表示フィルタ」で拠点別に絞り込み可能（全拠点/東京/大阪）
- 「完了」ステータスのデータは自動的に非表示

## データ構造

### スプレッドシート「OrderData」シート（V3スキーマ - 18列）

| 列名 | 列 | 説明 |
|------|-----|------|
| branch_name | A | 拠点名 |
| file_id | B | Google Drive ファイルID |
| file_name | C | ファイル名（編集可能） |
| status | D | 処理ステータス（未処理/処理中/確認中/完了/保留/要確認） |
| order_date | E | 発注日 |
| maker_name | F | メーカー名 |
| shop_name | G | 店舗名 |
| product_code | H | 品番 |
| product_name | I | 商品名 |
| unit_price | J | 単価 |
| quantity | K | 数量 |
| line_total | L | 小計（単価×数量） |
| processed_at | M | 処理日時 |
| delivery_destination | N | 納品先 |
| order_number | O | 発注番号 |

| item_order | Q | 商品明細の順序（PDF順） |

## カスタマイズ

### AIモデルの変更

`Code.gs` の `callGeminiApi()` 内の `MODEL_NAME` を変更（明細抽出用）：
```javascript
const MODEL_NAME = 'gemini-2.5-pro'; // または他のモデル
```

ページ数取得には `getPageCount()` 内の `gemini-2.5-flash` を使用しています（低コスト）。

### プロンプトのカスタマイズ

スプレッドシートの「プロンプト」シート A1セルを編集して、抽出項目や形式を変更できます。

### UIのカスタマイズ

`index.html` のスタイルやレイアウトを編集して、デザインを変更できます。

#### リサイザーの初期高さを変更

`.detail-section-top` のCSSで初期高さを調整：
```css
.detail-section-top {
  height: 420px; /* この値を変更 */
}
```

#### 商品明細No.列の幅を変更

テーブルヘッダーの`width`属性を変更：
```html
<th width="50px" style="text-align:center;">No.</th>
```

## トラブルシューティング

### PDFが処理されない場合

1. スクリプトプロパティが正しく設定されているか確認
2. Gemini API キーが有効か確認
3. Drive API が有効化されているか確認
4. フォルダ構造が正しいか確認（拠点フォルダ → PDFファイル）

### デバッグ方法

スクリプトエディタで以下を実行：
```javascript
debugFolderCheck();        // フォルダとファイルの診断
debugSchemaCheck();        // スプレッドシートのスキーマ診断
debugOcrWithDetailedLog(); // PDF読み取り結果の詳細ログ
purgeCompletedOrders();    // 完了レコードを手動削除
```

### エラーが発生したPDF

処理中にエラーが発生したPDFは、ファイル名に `【ERROR】` プレフィックスが付きます。
